# Changelog

Changes since v13.11.4:

## Highlights

- **Instant update checks** — the "Check for updates" button now always fetches fresh data from GitHub instead of returning cached results, and the startup update prompt always contacts the network when due. *(Hush)*
- **Thread‑safe Waze Bridge** — `readBundledBridge()` holds a lock around APK extraction to prevent concurrent writes corrupting the cached shim file. *(Hush)*
- **Saavn rate‑limit avoidance** — `findBestMatch()` runs search queries sequentially instead of launching up to 8 concurrent API calls, preventing JioSaavn from returning empty results. *(Hush)*
- **Car crash fixes** — `MusicService` no longer crashes on `getSystemService` returning `null` or when `onGetSession`/`onBind` is called before `onCreate` completes. *(Hush)*
- **Saavn defaults to off** — both `trySaavnFirst` and `EnableSaavnStreamingKey` default to `false` out of the box. *(Hush)*

## Source Resolution

- **Parallel source fetch primary/fallback** — when YouTube is the primary source (`trySaavnFirst=false`) and Saavn wins the race, the parallel path now waits the full grace period for YT instead of immediately canceling it; the serial fallback falls through to Saavn instead of throwing. *(Hush)*
- **Saavn timeouts reduced** — serial path timeout lowered from 15s to 8s, parallel path from 15s to 10s, grace period from 2.5s to 1.5s. *(Hush)*
- **Finding best match** — switched from parallel queries (8 concurrent) to sequential single‑query search to avoid JioSaavn rate limits. *(Hush)*

## Stability

- **Stale duplicate asset removed** — `app/src/main/assets/waze-shims.zip` deleted to eliminate confusion with the canonical copy at `app/src/mobile/assets/waze-shims.zip`. *(Hush)*
- **Null‑safe lateinit guards** — `onGetSession()` checks `::mediaSession.isInitialized`, `onBind()` checks `::player.isInitialized`, `onDestroy()` guards both; all `getSystemService` calls handle `null` with `stopSelf()`. *(Hush)*
- **Type‑safe system services** — `AudioManager`, `PowerManager`, `ActivityManager` use safe‑casts instead of raw `as`; `ConnectivityManager` uses `getSystemService<T>()` with explicit `?: error(...)`. *(Hush)*
- **Waze Bridge locking** — `readBundledBridge()` synchronized on `bridgeLock` to prevent APK file corruption under concurrent access. *(Hush)*

## Settings

- **Saavn defaults changed** — YouTube preferred by default (`trySaavnFirst=false`), streaming disabled by default (`EnableSaavnStreamingKey=false`). *(Hush)*

## Housekeeping

- Version bumped to 13.11.5 (versionCode 155). *(Hush)*
- Waze Bridge shim bumped to versionCode 156. *(Hush)*
