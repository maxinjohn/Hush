# Changelog

Changes since v13.11.4:

## Library

- **Loading state** — liked-songs page shows a spinner instead of flashing "0 songs" while Room query loads; refresh wrapped in 15s timeout so hanging sync can't freeze the UI indefinitely. *(Hush)*
- **Downloaded songs as playable list** — the Downloaded route now renders a proper playable list (hero artwork, song count, Play/Shuffle header) identical to Liked Songs, instead of redirecting to the download manager. *(Hush)*
- **Press animation** — play button in library spotlight card uses `hushCombinedPressable` for instant press-scale feedback. *(Hush)*
- **Cached songs loading** — cache playlist shows a loading indicator while initializing; stops polling after 10s if no cached songs found. *(Hush)*

## CI/CD

- **Missing overlay function** — `updateWebPoTokensIfSessionMatches` and its `authStateLock` were missing from the overlays/core YouTube.kt variant, causing GMS release builds to fail. Fixed. *(Hush)*
- Full parity verified between `core/` and `overlays/core/` — all 72 public functions and 22 properties present in both variants.
