# Changelog

Changes in v13.9.0..v13.10.0:

## Features

- **Waze Bridge** — Hush Music appears inside Waze's in-car audio player by impersonating Spotify and YouTube Music via companion shim APKs. Both shims can be installed simultaneously, and metadata updates are sent to all installed shims. *(Hush)*
- **JioSaavn Match Caching** — search results cached per video ID (up to 5000 entries) eliminates redundant API calls on repeat plays; stream URLs cached with 24-hour TTL avoids repeated DES decryption. *(Hush)*
- **Waze Like/Download Controls** — Like (thumbs up) and Download buttons available in Waze transport controls alongside play/pause/next/prev. *(Hush)*
- **Direct APK Updates** — tapping Update in the auto-detection bottom sheet now downloads the APK directly with a progress dialog instead of navigating to the update settings page. *(Hush)*
- **Download Pre-Resolution** — stream URLs are resolved asynchronously immediately when a download is added, preventing ExoPlayer's download manager from blocking and keeping progress visible. *(Hush)*
- **Waze Deferred Commands** — if Waze sends a command before the player queue is restored, it's saved and replayed after restore completes. *(Hush)*
- **Waze Next/Previous While Paused** — Waze playback controls work even when paused. *(Hush)*

## Player

- **Local cache priority** — downloaded songs play from cache regardless of JioSaavn setting; smart quality switching prefers cache at ≥160kbps or online at 320kbps when available. *(Hush)*
- **Waze empty queue fallback** — commands with 0 media items fall back to `play()` instead of a no-op seek. *(Hush)*
- **Waze periodic position sync** — playback position synced to Waze every second during active playback. *(Hush)*

## Settings

- **Waze Integration** — dedicated Waze Bridge section with independent install/remove for each shim, real-app conflict detection, and instant UI refresh. *(Hush)*

## Fixes

- **JioSaavn random song playing** — loose duration tolerance tightened from ±20s to ±10s; exact-title fallback now also checks duration. *(Hush)*
- **JioSaavn slow playback start** — match caching eliminates 3-4 redundant API searches per song on repeat plays. *(Hush)*
- **Download progress stuck** — pre-resolution of stream URLs prevents the download thread from blocking on URL resolution. *(Hush)*
- **Waze metadata sent to real apps** — metadata updates only sent to shims with matching package name and version code ≤2. *(Hush)*
- **SdkService binding leak** — duplicate Waze SdkService connections properly unbound. *(Hush)*
- **Foreground service race** — `ensureForeground()` synchronized to prevent Android 15+ crash. *(Hush)*
- **Waze `play_pause` command** — was silently dropped, now correctly forwarded from Waze intent receiver. *(Hush)*
- **Waze command delivery** — switched from `sendBroadcast` to `startService` with explicit `ComponentName` for guaranteed delivery. *(Hush)*
- **Shim launcher icon removed** — shim APKs have no app drawer entry. *(Hush)*
- **Update notification navigation** — notification now navigates to update screen instead of hanging. *(Hush)*
- **`getParcelableExtra` deprecation** — typed API for Tiramisu+, fallback for older versions. *(Hush)*
- **Waze cold start play** — pressing play in Waze when Hush is closed now restores the last queue and starts playback, instead of doing nothing. *(Hush)*
- **Waze next/previous metadata race** — next/previous while paused now correctly updates song metadata and position in Waze immediately. *(Hush)*
- **Waze play/pause progress freeze** — position now updates every second during playback, so Waze always shows correct progress. *(Hush)*
- **Waze Saavn duration hints** — duration now included in Saavn match hints for faster, more accurate JioSaavn stream resolution. *(Hush)*

## Housekeeping

- "Waze Integration" renamed to "Waze Bridge" in all user-facing strings. *(Hush)*
- Build scripts updated — fresh waze shim builds on every CI run. *(Hush)*
- Shim display names changed to "Spotify Bridge" / "YouTube Music Bridge". *(Hush)*
- Waze settings no longer shows package name for each shim. *(Hush)*
- Waze description no longer hardcodes "both" bridges. *(Hush)*
