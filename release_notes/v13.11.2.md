# Changelog

Changes in v13.9.0..v13.11.2:

## Features

- **Waze Bridge Updates** — installed Spotify and YouTube Music Bridges are checked against the embedded Bridge APK version, protocol, package identity, and signing certificate. Safe in-place updates are recommended only when compatible and newer. *(Hush)*
- **Waze Bridge Branding** — Bridges now appear as Hush (Spotify Bridge) and Hush (YouTube Music Bridge), using the Hush logo. *(Hush)*
- **Waze Bridge** — Hush Music appears inside Waze's in-car audio player by impersonating Spotify and YouTube Music via companion Bridge APKs. Both Bridges can be installed simultaneously, and metadata updates are sent to all installed Bridges. *(Hush)*
- **JioSaavn Match Caching** — search results cached per video ID (up to 5000 entries) eliminates redundant API calls on repeat plays; stream URLs cached with 24-hour TTL avoids repeated DES decryption. *(Hush)*
- **Direct APK Updates** — tapping Update in the auto-detection bottom sheet now downloads the APK directly with a progress dialog instead of navigating to the update settings page. *(Hush)*
- **Download Pre-Resolution** — stream URLs are resolved asynchronously immediately when a download is added, preventing ExoPlayer's download manager from blocking and keeping progress visible. *(Hush)*
- **Waze Deferred Commands** — if Waze sends a command before the player queue is restored, it's saved and replayed after restore completes. *(Hush)*
- **Waze Next/Previous While Paused** — Waze playback controls work even when paused. *(Hush)*

## Player

- **Local cache priority** — downloaded songs play from cache regardless of JioSaavn setting; smart quality switching prefers cache at >=160kbps or online at 320kbps when available. *(Hush)*
- **Waze empty queue fallback** — commands with 0 media items fall back to `play()` instead of a no-op seek. *(Hush)*
- **Waze periodic position sync** — playback position synced to Waze every second during active playback. *(Hush)*

## Settings

- **Waze Bridge** — dedicated Waze Bridge section with independent install/remove for each Bridge, real-app conflict detection, and instant UI refresh. *(Hush)*

## Waze Bridge

- **Hidden app redirect** — Bridges stay out of the Android launcher while Waze can still open Hush directly. *(Hush)*
- **Authoritative playback synchronization** — Waze receives ordered Hush playback snapshots for Play, Pause, seek, buffering, track changes, duration, buffered position, speed, and progress. *(Hush)*
- **Paused track changes** — Next and Previous publish the newly selected track, paused state, position, and queue item immediately. *(Hush)*
- **Bridge reconnect** — a verified Bridge update starts the updated Bridge service so Waze can reconnect without changing playback behavior. *(Hush)*
- **Play Store redirect** — tapping Waze's audio player now opens Hush instead of falling through to the app list. *(Hush)*
- **Guaranteed command delivery** — play/next/prev/pause commands use an explicit foreground-service intent so the Bridge starts `MusicService` and delivers the command payload. *(Hush)*
- **Cold start recovery** — pressing play in Waze when Hush was closed now works correctly even after the queue has already been restored. *(Hush)*
- **Show list** — media browser returns a single "Hush Music" playable item instead of an empty list. *(Hush)*
- **Like button icon** — DOWNLOAD action removed; THUMBS_UP uses a heart vector drawable with Media3 icon hint extras. *(Hush)*
- **Like button activation** — added `onPlayFromMediaId` handler so tapping the list item triggers playback. *(Hush)*
- **Stale files removed** — deleted an obsolete media browser service which referenced non-existent classes. *(Hush)*

## Fixes

- **Waze Bridge install/uninstall reliability** — Bridge detection switched from fragile `versionCode <= 2` check to explicit meta-data flag with legacy label fallback; install and uninstall now poll PackageManager with state verification instead of a blind delay; uninstall uses `Intent.ACTION_UNINSTALL_PACKAGE` with proper `ActivityResultLauncher` instead of opening Settings. *(Hush)*
- **Waze Bridge state loss on rotation** — pending install/uninstall app state survives configuration changes via `rememberSaveable`; fallback resets `isProcessing` when the launcher returns without a pending app. *(Hush)*
- **Waze Bridge uninstall guard** — confirm dialog respects `isProcessing` flag, preventing double-invocation during concurrent operations. *(Hush)*
- **Waze Bridge temp file leak** — extracted companion APKs deleted from cache after the install intent completes. *(Hush)*
- **Waze Bridge hardcoded strings** — Bridge labels and status messages now use `stringResource` references, enabling translation. *(Hush)*
- **JioSaavn random song playing** — loose duration tolerance tightened from +/-20s to +/-10s; exact-title fallback now also checks duration. *(Hush)*
- **JioSaavn slow playback start** — match caching eliminates redundant API searches per song on repeat plays. *(Hush)*
- **Download progress stuck** — pre-resolution of stream URLs prevents the download thread from blocking on URL resolution. *(Hush)*
- **Waze metadata sent to real apps** — metadata updates only sent to bridge packages signed with Hush's certificate. *(Hush)*
- **SdkService binding leak** — duplicate Waze SdkService connections properly unbound. *(Hush)*
- **Foreground service race** — `ensureForeground()` synchronized to prevent Android 15+ crash. *(Hush)*
- **Waze `play_pause` command** — was silently dropped, now correctly forwarded from Waze intent receiver. *(Hush)*
- **Waze cold start play** — pressing play in Waze when Hush is closed restores the last queue and starts playback. *(Hush)*
- **Waze next/previous metadata race** — next/previous while paused correctly updates song metadata and position in Waze immediately. *(Hush)*
- **Waze play/pause progress freeze** — position updates every second during playback, so Waze shows correct progress. *(Hush)*
- **Waze Saavn duration hints** — duration included in Saavn match hints for faster, more accurate JioSaavn stream resolution. *(Hush)*

## Housekeeping

- Keystore files (`*.jks`, `*.keystore_bk`, `persistent-debug.keystore`) removed from git tracking and gitignored. *(Hush)*
- Spurious build directories cleaned up to fix R8 duplicate class errors. *(Hush)*
- Version bumped to 13.11.2 (versionCode 152). *(Hush)*
- "Waze Integration" renamed to "Waze Bridge" in all user-facing strings. *(Hush)*
- Bridge display names changed to "Spotify Bridge" / "YouTube Music Bridge". *(Hush)*
- Waze settings no longer shows package name for each Bridge or assumes both Bridges are installed. *(Hush)*
- Waze Bridge install/uninstall screen now uses localized string resources. *(Hush)*
