# Changelog

Changes since v13.11.5:

## Highlights

- **Mood chip fix for logged-in accounts** ‚Äî clicking Energize, Relax, Sleep, etc. now works correctly when signed in to YouTube. The browse request falls back to anonymous mode when the authenticated response uses an incompatible page structure. *(Hush)*
- **Mood content placement** ‚Äî when a mood chip is active, its content now appears right after Quick Picks at the top instead of being buried at the bottom. Normal order is restored when the chip is deselected. *(Hush)*
- **Deezer Waze Bridge** ‚Äî new bridge shim for Deezer (`deezer.android.app`) with full playback controls and metadata display on Waze. *(Hush)*
- **Backup retention** ‚Äî configurable daily, weekly, and before-update backup limits (1, 3, 5, 10, or 20). Retention prunes automatically on startup, on schedule, and when limits change. *(Hush)*
- **Saavn beta warning** ‚Äî a confirmation dialog now appears when enabling JioSaavn as the primary audio source, appearing on both the JioSaavn Settings and Stream Quality screens. *(Hush)*
- **Gen‚ÄëZ explore theme** ‚Äî the search, explore, and mood/genre pages now use a vibrant visual style with neon mesh gradients, glass‚Äëmorphism cards, and bolder typography while the rest of the app keeps its standard theme. *(Hush)*

## Waze Bridge

- **Deezer bridge** ‚Äî added `deezer` product flavor to the Waze shim module with application ID `deezer.android.app`. The bridge definition, CI build scripts, and APK packaging are all updated. *(Hush)*
- **Deezer metadata** ‚Äî `WazeTargetApp` enum now includes `DEEZER` so the main Hush app broadcasts playback metadata to the Deezer shim. *(Hush)*
- **Shim signing restored** ‚Äî Waze shim builds no longer fall back to the release keystore from `local.properties` when `HUSH_SHIM_*` environment variables are absent, preventing accidental signing-key mismatches with previously installed shims. *(Hush)*

## Playback

- **Better error messages** ‚Äî when both YouTube and JioSaavn sources fail, the error now includes the actual failure reason from each source (e.g., timeout, bot detection, no match) instead of a generic "Both sources failed" message. *(Hush)*

## DoH & Network

- **DoH cache recovery** ‚Äî failed DNS-over-HTTPS lookups now clear the cached delegate so the next lookup attempts a fresh connection. *(Hush)*
- **DoH diagnostics** ‚Äî the "Test DNS connection" button shows specific error messages (connection timeout, host not found, endpoint blocked) instead of a generic "Unknown error" fallback. *(Hush)*

## Home Screen

- **Mood chip auth fallback** ‚Äî `YouTube.home()` tries the browse request with login first, then retries anonymously if the authenticated response has no parseable sections. Three response structures are now supported: `contents.sectionListRenderer`, `singleColumnBrowseResultsRenderer.tabs`, and `twoColumnBrowseResultsRenderer.tabs`. *(Hush)*
- **Chip content ordering** ‚Äî `RemoteSections` composable extracted and rendered in two conditional places: right after Quick Picks when a mood chip is active, or at the bottom among the default YouTube carousels when no chip is selected. *(Hush)*

## Settings

- **Backup retention** ‚Äî new preference keys (`AutoBackupDailyRetentionKey`, `AutoBackupWeeklyRetentionKey`, `AutoBackupBeforeUpdateRetentionKey`) and `AutoBackupRetention` utility with default limit of 5 and selectable values 1/3/5/10/20. Retention UI is mutually exclusive per backup frequency; before‚Äëupdate retention appears separately when enabled. Pruning runs after each automatic backup write, on limit change, and at app startup. *(Hush), ported from Vivi Music*
- **Saavn beta warning** ‚Äî `SaavnBetaWarningDialog` composable shown when switching the primary audio source to JioSaavn from either `JioSaavnSettings` or `StreamQualitySettings`. The dialog explains the beta nature of the feature before confirming. *(Hush)*

## UI / Theme

- **Gen‚ÄëZ explore theme** ‚Äî `LocalExploreTheme` CompositionLocal, `HushExploreBackground` with higher‚Äëchroma neon blobs, `rememberHushVibrantGradient`, and `Modifier.hushExploreCard` for glass‚Äëmorphism cards with primary‚Äëtinted borders. Applied to `SearchScreen`, `MoodAndGenresScreen`, and mood/genre buttons. *(Hush)*
- **Mood button redesign** ‚Äî `MoodAndGenresButton` now uses bolder gradients (lerped against primary/tertiary instead of muted containers), a glass border when the explore theme is active, and `titleLarge ExtraBold` typography. *(Hush)*

## Housekeeping

- Version bumped to 13.11.7 (versionCode 157). *(Hush)*
- Waze Bridge shim bumped to versionCode 159. *(Hush)*
