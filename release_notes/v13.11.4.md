# Changelog

Changes since v13.11.3:

## Highlights

- **Parallel Source Fetch** ‚Äî resolve YouTube and JioSaavn streams simultaneously instead of waiting for one to fail before trying the other. Whichever responds first is used immediately, reducing playback start time. *(Hush)*
- **JioSaavn Performance Overhaul** ‚Äî search queries, API format fallback, and stream decryption paths now all run concurrently instead of sequentially, cutting latency from ~10s‚Äì30s to ~1s‚Äì3s. *(Hush)*
- **Local/Cache Priority** ‚Äî downloaded and cached songs always play from local storage without attempting remote streaming. *(Hush)*

## Player

- **Parallel source fetch toggle** under Audio Source ‚Üí Stream Quality settings; when enabled, YT and JioSaavn resolution race concurrently. *(Hush)*
- **AOD player z-index fix** ‚Äî always-on-display player rendered at top z-index to prevent normal-player visual overlap; network-status banner suppressed while AOD is active. *(Hush)*
- **Ringtone action** ‚Äî local MediaStore songs expose "Set as ringtone" in both Player and library SongMenu; WRITE_SETTINGS permission requested on first use. *(Hush)*
- **Prefetch semaphore** capped at 4 concurrent jobs (was 3); maximum prefetch count normalized to 4 (was 20). *(Hush)*
- **Stale prefetch cancel** ‚Äî queued lookahead deferreds cancelled when a new queue replaces the current one. *(Hush)*
- **Prefetch skip for cached songs** ‚Äî stream URL resolution no longer started for already-downloaded tracks. *(Hush)*

## JioSaavn

- **Parallel search** ‚Äî all search queries (album+title+artist, title+artist, title, artist) launched concurrently; first strict match wins, remaining queries cancelled. *(Hush)*
- **Parallel API format** ‚Äî v4 and legacy search API formats raced simultaneously instead of retrying sequentially. *(Hush)*
- **Parallel stream resolution** ‚Äî auth token, DES decrypt v4, and DES decrypt flat paths all run concurrently; first successful URL returned. *(Hush)*

## Settings

- **"Choose folder"** ‚Äî download/cache folder picker button renamed from "Import from folder" for clarity; success message uses "Folder selected" consistently. *(Hush)*
- **App language selector** ‚Äî packaged translations exposed with locale recreation on change (pre-Android 13). *(Hush)*

## Stability

- **Queue/radio race conditions** ‚Äî generation-based stale detection prevents duplicate adds, phantom tracks, and disabled radio state after queue replacement. *(Hush)*
- **Infinite queue generation guard** ‚Äî stale radio bootstrap discarded when queue or current track changes mid-load. *(Hush)*
- **Playback auth PoToken** ‚Äî atomic session-matched PoToken updates via `withMintedWebPoTokens`; BotGuard pre-warm uses session ID instead of visitor data. *(Hush)*
- **Canvas cache** ‚Äî prefers validated local MP4s over re-download; partial/corrupt files cleaned and trimmed on stale recovery. *(Hush)*
- **Quick Picks** ‚Äî sample count raised from 20 to 40 for better carousel image stability. *(Hush)*

## Removed

- **Compatible-apps scanner/auto-importer** ‚Äî removed entirely. Android blocks cross-app private-storage reads even with MANAGE_EXTERNAL_STORAGE. Users must export manually. *(Hush)*
- **MANAGE_EXTERNAL_STORAGE permission** and all competitor package queries removed from manifest. *(Hush)*
- **Legacy storage auto-import** ‚Äî `LegacyStorageAutoImporter`, `LegacyStorageCompatibility`, `StorageEntryPoint` DI entry deleted. *(Hush)*
- **Duplicate UI elements** ‚Äî redundant Account toolbar token/key button and duplicate nested scroll handler removed. *(Hush)*

## Housekeeping

- Debug log output (`pulse_debug.log`) removed from `PulseMatrixEngine`; all production Log/Timber calls guarded by `BuildConfig.DEBUG`. *(Hush)*
- Release keystore (`release.keystore`) removed from git tracking and added to `.gitignore`. *(Hush)*
- Waze Bridge shims rebuilt. *(Hush)*
- Version bumped to 13.11.4 (versionCode 154). *(Hush)*
